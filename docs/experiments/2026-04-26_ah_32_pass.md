# Experiment: A-H 32/32 Promotion

Date: 2026-04-26

## Question

Can the tutor promote a clean A-H checkpoint for `copy,copy2,next,next2` under a
strict reset-based `32/32` exam?

## Hypothesis

The best A-H near miss can be repaired, but the repair may shift failure to a
neighboring successor item. Strict promotion and near-miss saving are needed to
avoid accepting a partial checkpoint.

## Setup

- Clean A-G base: `checkpoints/kinder_ABCDEFG_alltasks_ctxn_v2.pt`
- Best diagnostic A-H base: `runs/abcdefgh_targeted_try_local_v5.pt`
- Targets: `ABCDEFGH`
- Tasks: `copy,copy2,next,next2`
- Device: CPU
- Generation mode: legacy restep generation enabled
- Final promotion gate: `32/32`, no regressions

## Protocol Summary

First strict sweep from the diagnostic A-H checkpoint:

```bash
.venv/bin/python scripts/tutor_guarded_runner.py --text-path input.txt \
  --checkpoint runs/abcdefgh_targeted_try_local_v5.pt \
  --out-dir runs/guarded_h_full_from_v5_strict_run2 \
  --targets ABCDEFGH --tasks copy,copy2,next,next2 \
  --rounds 1 \
  --steps-candidates 10,20,30,50 \
  --lr-candidates 0.00002,0.00005,0.0001 \
  --focus-mults 4,8,16,32 \
  --repair-attempts 2 --repair-steps-candidates 5,10 \
  --repair-lr-scale 0.5 --save-best-rejected
```

This found and saved a non-promoted `31/32` near miss:

```text
runs/guarded_h_full_from_v5_strict_run2/best_rejected.pt
```

The single failure was:

```text
next2: N:ABCDEFGH:EF:n -> F, expected G
```

Restep-aligned replay of that failure fixed `EF -> G` but shifted the failure to
`FG -> H`:

```text
runs/guarded_h_diagnostic_ef_fixed_fg_regressed/best.pt
```

Final strict repair from the shifted failure:

```bash
.venv/bin/python scripts/tutor_guarded_runner.py --text-path input.txt \
  --checkpoint runs/guarded_h_diagnostic_ef_fixed_fg_regressed/best.pt \
  --out-dir runs/guarded_h_from_fg_regression_strict_run1 \
  --targets ABCDEFGH --tasks copy,copy2,next,next2 \
  --rounds 1 \
  --steps-candidates 1,2,3,5,8,13,20 \
  --lr-candidates 0.000001,0.000002,0.000005,0.00001,0.00002,0.00005 \
  --focus-mults 0,1,2,4,8 \
  --maintenance-radius 0 \
  --failure-repeats 512 \
  --align-restep-training \
  --w-copy 1 --w-copy2 1 --w-next 1 --w-next2 1 \
  --repair-attempts 2 --repair-steps-candidates 1,2,5 \
  --repair-lr-scale 0.5 --save-best-rejected
```

## Observations

- The first useful near miss passed all copy/copy2 tasks and all but one
  successor item.
- Repairing `EF -> G` did not monotonically improve the whole concept battery.
  It shifted failure to `FG -> H`.
- The final clean pass required repairing the shifted failure under the full
  exam, not only replaying the original failed prompt.
- Restep-aligned training was necessary for targeted repair because the exam
  steps the final `:` before decoding.

## Interpretation

Observation: fixing one `next2` item can damage a neighboring `next2` item.

Inference: the successor behavior shares internal machinery across neighboring
items; it is not a set of independent prompt-answer slots.

Speculation: the model may be balancing a small recurrent successor/continuation
geometry rather than storing a clean symbolic rule.

## Alternative Explanations

- The effect may be specific to legacy restep generation.
- The final clean state may be a narrow local balance rather than a robust
  abstraction.
- The same result may not transfer across seeds or a differently initialized
  model.
